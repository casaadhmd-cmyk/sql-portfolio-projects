# Data Catalog – Gold Layer

## Overview

The Gold Layer represents the **business-level data model** designed for reporting and analytics.
This layer follows a **Star Schema** design, consisting of **Dimension tables** and a **Fact table**.

The purpose of the Gold Layer is to provide **clean, business-ready data** that can be used by analysts, reporting tools, and dashboards.

> Note: This project was implemented using **SQLite**, so data types such as TEXT, INTEGER, and REAL are used. Dates are stored in TEXT format (YYYY-MM-DD).

---

# 1. gold.dim_customers

**Purpose:**
Stores customer master data enriched from CRM and ERP systems. This dimension table provides customer demographic and geographic information for analysis.

**Table Type:** Dimension Table

| Column Name     | Data Type | Description                                             |
| --------------- | --------- | ------------------------------------------------------- |
| customer_key    | INTEGER   | Surrogate key uniquely identifying each customer record |
| customer_id     | INTEGER   | Unique customer ID from source system                   |
| customer_number | TEXT      | Customer reference number                               |
| first_name      | TEXT      | Customer first name                                     |
| last_name       | TEXT      | Customer last name                                      |
| country         | TEXT      | Country of residence                                    |
| marital_status  | TEXT      | Marital status of the customer                          |
| gender          | TEXT      | Gender of the customer                                  |
| birthdate       | TEXT      | Customer date of birth (YYYY-MM-DD)                     |
| create_date     | TEXT      | Record creation date                                    |

---

# 2. gold.dim_products

**Purpose:**
Stores product master data including product category and maintenance information. This table helps analyze sales performance by product, category, and product line.

**Table Type:** Dimension Table

| Column Name          | Data Type | Description                                     |
| -------------------- | --------- | ----------------------------------------------- |
| product_key          | INTEGER   | Surrogate key uniquely identifying each product |
| product_id           | INTEGER   | Unique product ID from source system            |
| product_number       | TEXT      | Product reference number                        |
| product_name         | TEXT      | Name of the product                             |
| category_id          | TEXT      | Product category ID                             |
| category             | TEXT      | Product category name                           |
| subcategory          | TEXT      | Product subcategory                             |
| maintenance_required | TEXT      | Indicates if maintenance is required            |
| cost                 | INTEGER   | Product cost                                    |
| product_line         | TEXT      | Product line (e.g., Road, Mountain)             |
| start_date           | TEXT      | Product availability start date                 |

---

# 3. gold.fact_sales

**Purpose:**
Stores transactional sales data. This is the central fact table used for revenue, quantity, and sales performance analysis.

**Table Type:** Fact Table

| Column Name   | Data Type | Description                  |
| ------------- | --------- | ---------------------------- |
| order_number  | TEXT      | Unique sales order number    |
| product_key   | INTEGER   | Foreign key to dim_products  |
| customer_key  | INTEGER   | Foreign key to dim_customers |
| order_date    | TEXT      | Date when order was placed   |
| shipping_date | TEXT      | Date when order was shipped  |
| due_date      | TEXT      | Payment due date             |
| sales_amount  | REAL      | Total sales amount           |
| quantity      | INTEGER   | Number of items sold         |
| price         | REAL      | Price per item               |

---

# Data Model (Star Schema)

**Fact Table:**

* fact_sales

**Dimension Tables:**

* dim_customers
* dim_products

**Relationships:**

* fact_sales.customer_key → dim_customers.customer_key
* fact_sales.product_key → dim_products.product_key

This structure forms a **Star Schema**, which is optimized for analytical queries and reporting.

---

# Data Flow Architecture

| Layer  | Description                                       |
| ------ | ------------------------------------------------- |
| Bronze | Raw data ingested from CRM and ERP source systems |
| Silver | Cleaned, standardized, and transformed data       |
| Gold   | Business-ready data modeled into Star Schema      |
